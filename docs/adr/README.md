# Architecture Decision Records

Significant, hard-to-reverse decisions are recorded here as ADRs (Michael Nygard style).

- One file per decision: `NNNN-slug.md` (zero-padded, monotonic — `0001-…`, `0002-…`).
- Copy `0000-template.md`, fill it in, set **Status** (Proposed → Accepted → Superseded).
- A superseded ADR stays (history); its Status links to the ADR that replaces it.

The template's sections are **Status / Context / Decision / Consequences**.
